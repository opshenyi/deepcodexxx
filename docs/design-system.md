# DeepCodex Design System

DeepCodex uses `D:\design-md` as the local design reference library. The current product UI borrows selectively from developer-tool brands rather than copying any single visual identity.

## Reference Sources

- Cursor: warm engineering canvas, restrained orange accent, code-heavy product surfaces.
- OpenCode: terminal-native rhythm, hairline sections, mono-friendly dense reading.
- Linear: high-contrast product console, precise panels, visible execution state.
- Vercel: shadow-as-border restraint, clean spacing, technical typography.
- Raycast: command-palette-like dark product chrome, compact event surfaces.
- Warp: calm tool atmosphere and low-decoration interaction.

## Applied Direction

- Canvas: warm neutral rather than cold SaaS blue.
- Primary color: near-black for core actions and text.
- Accent: a single DeepCodex orange used only for active state and running state.
- Borders: hairline containment with minimal elevation.
- Radius: 4px to 8px for functional controls and panels.
- Typography: system sans for UI, system mono for event bodies, no icon dependency.
- Layout: three-pane agent console with controls, event stream, and memory/output rail.
- Review surfaces: unified and split diff modes use the same restrained borders, mono text, and wrapping rules as the event timeline.

## Rules

- Do not use emoji or icon-led controls.
- Keep action labels textual and explicit.
- Avoid decorative gradients and floating visual effects.
- Prefer product data, event traces, and terminal-like content over marketing copy.
- Preserve mobile usability with single-column stacking and long-text wrapping.
